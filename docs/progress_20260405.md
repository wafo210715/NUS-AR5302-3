#### Key technical decisions
- **Bus code zero-padding on read**: OD data stores bus codes as numeric (e.g. "05239" → R reads as 5239). `pad_bus_code()` restores leading zeros via `sprintf("%05d", as.numeric(code))` before matching against `station_mapping.pt_code`.
- **Train codes**: No padding needed — OD data stores them as strings (e.g. "CC24", "EW12/DT14").
- **Filter strategy (OD)**: Keep rows where **either** `ORIGIN_PT_CODE` **or** `DESTINATION_PT_CODE` matches a campus station code.
- **Filter strategy (node)**: Keep rows where `PT_CODE` matches a campus station code.
- **`eval=FALSE`**: Extraction reads ~18 GB of data across 296 files. Not suitable for repeated knitting. Run chunks manually in R console once, then results persist on disk.
- **Output directories**: `data/od_subset/` and `data/node_subset/` created by the script. Both are gitignored (derived data).
- **Quality checks**: Row counts per month, missing month detection (expects all 74 months: Jan 2020 – Feb 2026), station coverage (how many of the 136 campus stations appear per month), anomaly flagging (>2 SD from mean row count).

#### Data quality checks chunk details
- `build_quality_df()`: Generic function that scans subset files and returns a tibble with `file`, `year_month`, `total_rows`, `total_trips`, `stations_found`, `stations_missing`.
- Separate bus/train handling via `is_train` flag (controls zero-padding).
- Final report combines all 4 datasets into one CSV with a `dataset` column.
