# Part 4: Time-of-Day POI Analysis Proposal

## 概述

不依赖opening_hours标签（覆盖率仅8%），而是利用OD数据的时间段信息，分析不同时段的站点功能特征。

## 数据准备

### 1. OD数据时间段划分

从OD数据中提取出行时间，定义三个时段：

| 时段 | 时间范围 | 出行类型 |
|------|----------|----------|
| 早高峰 | 06:00-10:00 | 上班、上学 |
| 日间 | 10:00-16:00 | 商务、休闲 |
| 晚高峰 | 16:00-22:00 | 下班、娱乐 |

### 2. POI-时段关联策略

**方案A：基于出行目的推断**
- 早高峰POI权重：学校、办公室、交通枢纽
- 日间POI权重：餐饮、零售、商业
- 晚高峰POI权重：娱乐、餐饮、住宅区

**方案B：基于OD流量加权**
- 计算每个站点在三个时段的到达/离开流量
- 用流量作为权重，重新计算站点-POI矩阵
- 对每个时段运行独立的LDA
