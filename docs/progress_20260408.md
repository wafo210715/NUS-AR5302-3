## 三、Topic Coherence计算（完整版）

### 3.1 问题

**原方案：** 简化版coherence计算（词汇频率模拟）
**问题：** 不是真正的C_v coherence，无法支持K值选择

### 3.2 解决方案

**Step 1：** `utils/export_lda_beta.R` - 从R导出beta矩阵

```r
# 导出每个K值的beta矩阵
model <- readRDS("data/lda_model_k5.rds")
beta <- posterior(model)$terms  # topics x words
write.csv(beta, "data/lda_beta/beta_k5.csv")
```

**Step 2：** `utils/calculate_topic_coherence_gensim.py` - 用gensim计算C_v

### 3.3 Coherence结果（真实）

| K值 | C_v Coherence | U_MASS | 排名 |
|-----|--------------|--------|------|
| **K=5** | **0.5063** | -2.1045 | **1** ← 最优 |
| K=8 | 0.5027 | -2.1679 | 2 |
| K=7 | 0.4965 | -2.1710 | 3 |
| K=6 | 0.4831 | -2.1212 | 4 |
| K=4 | 0.4611 | -2.1304 | 5 |

**结论：** **K=5确认是最优选择**，与perplexity+领域知识判断一致

**输出文件：**
- `data/lda_beta/beta_k4.csv` ... `beta_k8.csv` - Beta矩阵
- `data/coherence_scores_gensim.json` - Coherence分数

## 四、Part 4: POI Metadata与Opening Hours分析

### 4.1 三个关键问题的答案

#### Q1: LDA基于什么数据？

**答案：所有有POI键的数据，不依赖name字段**

- LDA输入：56,627个POI
- 特征：`poi_class`（如`amenity=restaurant`）
- `name`仅用于质量指标计算（53.1%有名称）
- 不过滤无名称的POI

#### Q2: Part 4时间段分析策略

**方案：** 使用有opening_hours的POI，与OD时间对齐

| 时段 | 时间范围 | POI数量 | 覆盖率 |
|------|----------|--------|--------|
| 早高峰 | 06:00-10:00 | 2,252 | 50.3% |
| 日间 | 10:00-16:00 | 4,197 | **93.8%** ✓ |
| 晚高峰 | 16:00-22:00 | 4,056 | **90.7%** ✓ |

**总POI：** 4,473个（7.9%覆盖率）

#### Q3: 可用Metadata字段

| 字段 | Part 4用途 |
|------|-----------|
| `version` | 质量过滤（排除v1占比高的站点） |
| `timestamp` | 编辑时间 |
| `user_id` | 贡献者数量 |
| `amenity`, `shop`, `leisure`... | POI主类别 |
| `lat`, `lon` | 空间位置 |
| `all_tags` | 完整标签（含opening_hours） |
